# Programmering eksamen - overblik

De her noter er tænkt som hurtige huskenoter. Ikke fuld teori, bare de ting der er gode at kunne finde hurtigt under forberedelse eller eksamen.

Typisk workflow i vores projekter:

```sh
cmake -S . -B build
cmake --build build
./build/program
```

Hvis der er tests:

```sh
cmake -S . -B build
cmake --build build
./build/tests
```

Hvis projektet har scripts:

```sh
./run_pub.sh
./run_sub.sh
./run_catch.sh
```

Gode ting at tjekke når noget fejler:

- Er du i den rigtige mappe?
- Findes `CMakeLists.txt` i projektroden?
- Er `build/` blevet lavet?
- Hedder executable det samme som target i `CMakeLists.txt`?
- Mangler du et library i `target_link_libraries(...)`?
- Kører MQTT broker på `localhost:1883`?
- Har shell scriptet execute permission? `chmod +x script.sh`
